# Ory Polis — SAML/OIDC Bridge

> **Docker image:** Uses `boxyhq/jackson` (public) — Ory Polis is built on BoxyHQ SAML Jackson
> and does not have a publicly available Docker image. The image, env vars, and API are identical.

Polis acts as the bridge between your IAM system and GitLab:

- **Toward GitLab** → Polis is a SAML Identity Provider (IdP)
- **Toward Hydra** → Polis is an OIDC Relying Party (RP)

---

## Architecture & Authentication Flow

```
Browser
  │
  │  1. User clicks "Open GitLab" in IAM app
  ▼
Ory Polis  (IdP-initiated SSO URL)
  │
  │  2. Polis starts OIDC Authorization Code flow
  ▼
Ory Hydra  (OAuth2 / OIDC Provider)
  │
  │  3. Hydra sends login_challenge to IAM app
  ▼
IAM App  (Next.js)
  │
  │  4. IAM app checks Kratos session via /sessions/whoami
  │     ├─ Session exists → silently accept login + consent (SSO, no prompt)
  │     └─ No session    → show login form → user logs in → repeat from step 3
  ▼
Ory Hydra
  │
  │  5. Issues ID token with: email, name, preferred_username
  ▼
Ory Polis
  │
  │  6. Translates OIDC claims → signed SAML Assertion
  │     POSTs SAMLResponse to GitLab ACS URL
  ▼
GitLab (rexform-sandbox group)
  │
  └─ Validates assertion → creates session → user is logged in ✓
```

## Environment Variables

### Required

| Variable | Description | How to get it |
|---|---|---|
| `POLIS_DB_URL` | PostgreSQL connection string for Polis | Create a Postgres service in Railway, copy `DATABASE_URL` |
| `POLIS_EXTERNAL_URL` | Public HTTPS URL of this Polis service | Railway-generated domain |
| `POLIS_NEXTAUTH_SECRET` | Signs Polis internal session cookies | `openssl rand -hex 32` |
| `POLIS_API_KEYS` | Protects the Polis admin API | `openssl rand -hex 32` |
| `POLIS_SAML_PRIVATE_KEY` | Polis signs SAML assertions with this | Generated by openssl (see below) |
| `POLIS_SAML_PUBLIC_KEY` | Matching certificate — pasted into GitLab | Generated by openssl (see below) |
| `POLIS_HYDRA_CLIENT_ID` | OAuth2 client ID registered in Hydra | `polis-saml-bridge` |
| `POLIS_HYDRA_CLIENT_SECRET` | OAuth2 client secret registered in Hydra | `openssl rand -hex 32` |
| `HYDRA_ISSUER` | Hydra public URL — Polis fetches OIDC discovery from here | Your Hydra Railway domain |
| `PORT` | Internal port Polis listens on | `5225` |

### Optional (SMTP — only needed for email notifications)

| Variable | Description |
|---|---|
| `SMTP_HOST` | SMTP server hostname |
| `SMTP_PORT` | SMTP port (default: 587) |
| `SMTP_USER` | SMTP username |
| `SMTP_PASSWORD` | SMTP password |
| `SMTP_FROM` | From address for emails |

## Setup Steps

### 1. Generate SAML signing certificate

Polis signs every SAML assertion it sends to GitLab with this certificate.

```bash
openssl req -x509 -newkey rsa:2048 -keyout polis-saml.key -out polis-saml.crt \
  -days 3650 -nodes \
  -subj "/CN=polis-production-1f3a.up.railway.app/O=Rexform/C=US"
```

Convert to single-line format for Railway env vars (newlines replaced with `|`):

```bash
# Paste output as POLIS_SAML_PRIVATE_KEY
awk 'NF {printf "%s|", $0} END {print ""}' polis-saml.key

# Paste output as POLIS_SAML_PUBLIC_KEY
awk 'NF {printf "%s|", $0} END {print ""}' polis-saml.crt

# Get SHA256 fingerprint — paste into GitLab "Certificate fingerprint" field
openssl x509 -noout -fingerprint -sha256 -in polis-saml.crt
```

### 2. Register Polis as an OAuth2 client in Hydra

Run once against your Hydra admin API:

```bash
curl -X POST https://hydra-production-a56f.up.railway.app/admin/clients \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "polis-saml-bridge",
    "client_name": "Ory Polis SAML Bridge",
    "client_secret": "YOUR_POLIS_HYDRA_CLIENT_SECRET",
    "grant_types": ["authorization_code", "refresh_token"],
    "response_types": ["code"],
    "scope": "openid email profile offline_access",
    "redirect_uris": ["https://polis-production-1f3a.up.railway.app/api/oauth/callback"],
    "token_endpoint_auth_method": "client_secret_post",
    "subject_type": "public",
    "skip_consent": false
  }'
```

> `skip_consent` must be `false` so Hydra calls the IAM app consent handler,
> which injects email and username into the ID token for Polis to use.

### 3. Deploy Polis on Railway

- Create a new Railway service from the `polis/` directory
- Set port to `5225` when generating the domain
- Add all required env vars from the table above
- Deploy — Railway builds from the Dockerfile

### 4. Register GitLab as a SAML Service Provider in Polis

Run after Polis is deployed and healthy:

```bash
curl -X POST https://polis-production-1f3a.up.railway.app/api/v1/connections \
  -H "Authorization: Api-Key YOUR_POLIS_API_KEYS_VALUE" \
  -H "Content-Type: application/json" \
  -d '{
    "defaultRedirectUrl": "https://gitlab.com/groups/rexform-sandbox",
    "redirectUrl": ["https://gitlab.com/groups/rexform-sandbox/*"],
    "tenant": "rexform-sandbox",
    "product": "gitlab-saml",
    "name": "GitLab Rexform Sandbox",
    "oidcDiscoveryUrl": "https://hydra-production-a56f.up.railway.app/.well-known/openid-configuration",
    "oidcClientId": "polis-saml-bridge",
    "oidcClientSecret": "YOUR_POLIS_HYDRA_CLIENT_SECRET"
  }'
```

### 5. Configure GitLab group SAML

Go to **GitLab → rexform-sandbox → Settings → SAML SSO** and fill in:

| GitLab field | Value |
|---|---|
| Identity provider single sign-on URL | `https://polis-production-1f3a.up.railway.app/api/v1/saml/sso?tenant=rexform-sandbox&product=gitlab-saml` |
| Certificate fingerprint | SHA256 output from step 1 |

### 6. Add the GitLab button in your IAM app

```tsx
<a href="https://polis-production-1f3a.up.railway.app/api/v1/saml/idp/sso?tenant=rexform-sandbox&product=gitlab-saml">
  Open GitLab
</a>
```

## OIDC → SAML Claim Mapping

| Kratos Identity Trait | Hydra ID Token Claim | SAML Attribute (sent to GitLab) |
|---|---|---|
| `traits.email` | `email` | `email` |
| `traits.username` | `preferred_username` | `preferred_username` |
| `traits.name.first` + `.last` | `name` | `name` |
| identity ID | `sub` | NameID (emailAddress format) |

## Common Issues

| Symptom | Cause | Fix |
|---|---|---|
| Infinite redirect loop | Kratos session cookie not forwarded to IAM app | Ensure `cookie` header is passed to `kratosPublic.toSession()` |
| `redirect_uri_mismatch` from Hydra | Polis callback URL not matching registered client | Verify `redirect_uris` in Hydra client matches `POLIS_EXTERNAL_URL/api/oauth/callback` |
| GitLab rejects SAML assertion | Cert mismatch | Regenerate cert and update both `POLIS_SAML_PRIVATE_KEY` and GitLab fingerprint together |
| ID token missing email/name | `skip_consent: true` on Hydra client | Set `skip_consent: false` so consent handler runs |
| Polis fails to start | Keys have real newlines instead of `\|` | Re-run the `awk` commands and update Railway env vars |
